Hive - Demo Video Production Sheet

Everything to shoot, narrate & edit the ~90-second hackathon demo. OBS + iPhone Screen Recording + YarnGPT.

Target: 90s–2:00 1080p landscape Voiceover: YarnGPT Record: OBS Studio
The one rule: show the product doing real work, in real time. The moment that wins is a buyer pays on WhatsApp → the merchant dashboard updates live on the same screen. Build the whole video around landing that shot.

1 Gear & tools

Screen recordOBS Studio (free) - the whole video.
iPhone screenBuilt-in Screen Recording (cleanest), or AirPlay to Windows for a live split.
VoiceoverYarnGPT - paste the script (§5), export the audio.
Edit + captionsDescript or CapCut - assemble, auto-caption, trim dead air.

On iPhone + Windows you can't use scrcpy (Android only). Instead, record the WhatsApp flow with the iPhone's built-in Screen Recording and drop that clip into the editor - perfect quality, zero mirroring. See §3.

2 OBS Studio setup

Output settings (Settings → Output → Recording)

  • Recording format: MP4
  • Encoder: hardware (NVENC / QuickSync) if available, else x264
  • Rate control: CBR, Bitrate: 12000–16000 Kbps

Video settings (Settings → Video)

  • Base & Output resolution: 1920×1080
  • FPS: 30

What OBS records (the desktop half)

  • Scene A - Landing: a Display/Window Capture of the browser on hive-nomba-web.vercel.app.
  • Scene B - Dashboard: Window Capture of the merchant dashboard on store Bella's Fashion Hub. This is what you'll film while the payment lands, so the revenue/orders visibly tick up.

Your iPhone WhatsApp footage is recorded separately on the phone (§3). The "split-screen money shot" is assembled in the editor - phone clip on the left, dashboard clip on the right - see §7. This is more reliable than live AirPlay mirroring and looks identical to viewers.

Audio tip: record video only in OBS (mute the mic) and lay the YarnGPT voiceover over it in the editor. Cleaner than narrating live, and you can re-do a line without re-shooting.

3 Capture your iPhone screen

Recommended - built-in Screen Recording (no mirroring, perfect quality)

  1. Enable it once: Settings → Control Center → add "Screen Recording".
  2. Swipe down into Control Center, long-press the record button, and turn the mic off (you'll add YarnGPT audio later). Tap Record.
  3. Perform the WhatsApp flow - seller adds a product, buyer orders, pays on the Nomba page. Take your time; you'll trim later.
  4. Stop the recording. Transfer the .mov to your PC via iCloud, a USB cable, or email to yourself, and drop it into the editor as the "phone" layer.
Why this over mirroring: it's rock-solid, full resolution, and needs no extra software. The only thing you give up is a live split screen - which you rebuild in the editor (§7) by placing the phone clip and the dashboard clip side by side. Viewers can't tell the difference.

Optional - live AirPlay to Windows (only if you want a true single-take split)

  • Install an AirPlay receiver on the PC: LetsView or 5KPlayer (free), or ApowerMirror (paid, most reliable).
  • Put phone + PC on the same Wi-Fi → iPhone Control Center → Screen Mirroring → pick the PC app.
  • The iPhone appears in a window; capture it in OBS as a Window Capture next to the dashboard for a real-time split.
  • Heads-up: AirPlay-to-Windows can stutter or drop. Test it first; if it's flaky, fall back to the built-in recorder above - it always works.

4 Pre-flight checklist

Before you hit record

  • Warm up the API - open the app once so Render isn't cold-starting mid-demo.
  • Seed data is live - dashboard should already read ~₦819k revenue, full chart. If not: node --env-file=apps/api/.env apps/api/scripts/seed-demo-orders.mjs
  • Two WhatsApp identities ready: your phone = merchant, a second number (or the Twilio sandbox) = buyer.
  • Buyer number has sent join <sandbox-code> to the Twilio sandbox so replies flow.
  • One full dry run end-to-end so the payment completes cleanly.
  • iPhone: Screen Recording added to Control Center, mic off, Do Not Disturb on (no banners mid-take).
  • Close notifications / Slack / email popups. Clean browser (no bookmarks bar clutter).
  • Dashboard open on the store Bella's Fashion Hub in OBS, ready to film while the payment lands.

8 Edit & export

  1. Drop three things into Descript (or CapCut): the iPhone screen recording, the OBS dashboard recording, and the YarnGPT audio. Line the voiceover up to the beats above.
  2. Build the money shot (1:16): place the phone clip and the dashboard clip side by side on screen at the payment moment, and nudge their timing so the dashboard revenue/orders tick up right as the chat says "confirmed." This is your fake-but-flawless split screen.
  3. Auto-generate captions - many judges watch muted. Keep them on the whole time.
  4. Add short on-screen labels at each beat: "Adding a product", "AI writes the listing", "Buyer pays via Nomba", "Stock updates live".
  5. Cut every spinner / cold-start lag / typo. No dead air - keep it moving.
  6. Optional: soft background music at ~15% under the voice. Never louder than the narration.
  7. Export: MP4, 1080p, 30fps. Aim for under 2:00.
Say "Nomba" out loud and put it on screen at the payment moment - it's their hackathon; make the sponsor integration unmissable. End on the live URL so judges can click through.
